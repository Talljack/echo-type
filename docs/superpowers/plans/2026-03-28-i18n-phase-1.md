# EchoType I18n Phase 1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a client-side English/Chinese i18n system for EchoType, with browser-based first-run language detection, explicit persisted language choice, and phase-1 migration of sidebar, dashboard, and settings.

**Architecture:** Extend the existing Zustand language store into a full preference model, add a typed translation dictionary plus a shared translator hook/provider, gate app-shell rendering until language initialization completes, then migrate app-owned copy on the sidebar, dashboard route, and settings route to the shared translation source.

**Tech Stack:** Next.js App Router, React 19, TypeScript, Zustand, Vitest, Playwright

---

## 1. FSRS 间隔重复

**算法:** [FSRS (Free Spaced Repetition Scheduler)](https://github.com/open-spaced-repetition/ts-fsrs) — Anki 5 最新采用的算法。

**工作原理:**
- 每次练习后，系统根据准确率自动映射到 FSRS Rating（Again/Hard/Good/Easy）
- Review 页面完成练习后，用户可手动选择评级，覆盖自动评级
- 每个评级按钮显示预测的下次复习时间（如 "Good · 4d"）

**关键文件:**
- `src/lib/fsrs.ts` — FSRS 调度器封装
- `src/components/review/rating-buttons.tsx` — 评级按钮组件
- `src/lib/daily-plan-progress.ts` — 练习后自动调度

**评级映射:**
| 准确率 | FSRS Rating |
|--------|-------------|
| < 50%  | Again       |
| 50-70% | Hard        |
| 70-90% | Good        |
| > 90%  | Easy        |

## 3. 发音评估

**双通道架构:**
```
用户语音 → 录音 (MediaRecorder API)
                ↓
    SpeechSuper API (优先)
    ↓ 配额用完或未配置
    AI 分析 (用户已有 AI provider)
                ↓
    统一的 PronunciationResult
                ↓
    音素显示 + IPA + 改进建议
```

**配置 SpeechSuper (可选):**
1. 注册 [SpeechSuper](https://www.speechsuper.com/) 获取 App Key 和 Secret Key
2. 在 Settings → Pronunciation 中填入
3. 免费额度：1000 次/月

**AI 兜底:** 无需额外配置，使用用户已配置的 AI provider 分析发音差异。

**关键文件:**
- `src/lib/pronunciation/` — 类型、SpeechSuper 客户端、AI 兜底
- `src/components/speak/phoneme-display.tsx` — 音素可视化
- `src/components/speak/pronunciation-tips.tsx` — 改进建议
- `src/stores/pronunciation-store.ts` — 配置存储

## 4. 登录认证 & 云同步

详见 [Supabase 配置指南](./supabase-setup.md)。

**架构:**
- **Auth:** Supabase Auth (Google/GitHub OAuth)
- **Sync:** 双向同步，last-write-wins 冲突解决
- **离线优先:** Supabase 未配置时 app 完全正常运行，所有数据存本地 IndexedDB

**同步的表:**
| Dexie 表 | Supabase 表 | 字段映射 |
|----------|-------------|---------|
| contents | contents | camelCase ↔ snake_case |
| records  | records  | 含 fsrsCard → fsrs_card |
| sessions | sessions | 含 startTime → start_time |

**关键文件:**
- `src/lib/supabase/` — 客户端（browser、server、middleware）
- `src/lib/sync/` — 同步引擎 + 字段映射器
- `src/stores/auth-store.ts` — 认证状态
- `src/stores/sync-store.ts` — 同步状态（30s 自动增量同步）
- `src/app/(app)/login/page.tsx` — 登录页
- `src/components/auth/user-menu.tsx` — 用户菜单
